# Personal Website

Portfolio website of Yuwei Li — built with Next.js, TypeScript, Tailwind CSS, and Sanity CMS.

Live at [yuweidesign.com](https://www.yuweidesign.com).

## Tech Stack

- **Next.js 16** (App Router, ISR) + **React 19**
- **TypeScript** — strict mode; CMS types generated by Sanity TypeGen
- **Tailwind CSS 3** — token-driven styling (see Design Tokens below)
- **Sanity CMS v4** — embedded Studio at `/studio`
- **GSAP** (`@gsap/react`, ScrollTrigger) — entrance / scroll animations
- **motion** — used by the vendored `DecryptedText` component
- **next-view-transitions** — page transition animations

```text
│  ├─ page.tsx                 # Home (fetches projects/skills/resume from Sanity)
│  ├─ projects/[slug]/page.tsx # Project detail pages (ISR, generateStaticParams)
│  ├─ studio/[[...index]]/     # Embedded Sanity Studio
│  ├─ sitemap.ts / robots.ts   # SEO
│  └─ opengraph-image.tsx / apple-icon.tsx  # Generated share/app icons
├─ components/
│  ├─ layout/                  # Navbar (StaggeredMenu wrapper)
│  ├─ projects/                # Project cards, gallery lightbox
│  ├─ providers/               # View-transitions provider
│  ├─ sections/                # Page sections (hero, about, projects, resume)
│  ├─ seo/                     # JSON-LD structured data
│  ├─ ui/                      # Reusable UI + animation primitives
│  │                           #   (Button, ScrollReveal, StaggerReveal,
│  │                           #    RevealText, Parallax, SkillsMarquee, …)
│  └─ vendor/                  # Third-party React Bits components (.jsx)
│                              #   + hand-written .d.ts type declarations
├─ hooks/                      # useHeroAnimation (GSAP timeline)
├─ lib/                        # Sanity client (server-only), cn() utility
├─ sanity/
│  ├─ lib/queries.ts           # ALL GROQ queries live here (defineQuery)
│  ├─ schemaTypes/             # project / resume / skillCategory schemas
│  └─ sanity.types.ts          # Generated by TypeGen — do not edit by hand
├─ types/                      # Type aliases derived from generated query results
└─ public/                     # Static assets (logo, hero svg, portrait)
```

## Conventions

- **Data flow**: pages (Server Components) fetch via `lib/sanity.ts` using queries
  from `sanity/lib/queries.ts`, then pass typed props into section components.
  Filtering (`visibility != false`) and ordering happen **in GROQ only** — never
  re-filter/re-sort in components.
- **Types**: never hand-write CMS types. Run `npm run typegen` after changing a
  schema or query; `types/*.ts` only re-exports aliases of generated result types.
- **Theming**: the site is single-theme (no dark mode toggle). The
  `design-light-*` / `design-dark-*` Tailwind colors are fixed *surface* tokens:
  light sections use `design-light-*`, the dark Work/detail sections use
  `design-dark-*`. Dark sections declare `data-overscroll-dark` so the body
  background matches during overscroll bounce.
- **Design tokens**: raw values live as CSS variables in `app/globals.css`,
  mapped to Tailwind utilities in `tailwind.config.ts`
  (spacing: `container/section/card/gap-*`, radius: `card/button/tag/media/panel`,
  motion: `duration-fast/base/slow/media`, type scale: `display/section/card/body/small/label`).
  Rule of thumb: a visual decision used 3+ times becomes a token; one-off art
  direction (portrait offsets, collage ratios) stays inline.
- **Owned interactive components**: `components/layout/staggered-menu.tsx` and
  `components/projects/masonry.tsx` are React Bits adaptations that now own core
  site behavior, so their prop contracts live in TSX source rather than handwritten
  `.d.ts` files.
- **Vendor components** (`components/vendor/`): low-touch React Bits visual
  components stay in JavaScript with local adaptation notes. Import DOM-heavy
  components dynamically with `ssr: false`.
- **Animations**: use the primitives in `components/ui/` (ScrollReveal,
  StaggerReveal, RevealText, Parallax). All of them respect
  `prefers-reduced-motion`; new animations must too.

## Setup

### Prerequisites

- Node.js 20/22 LTS and npm 10+
  (the Sanity CLI — used by `npm run typegen` — currently fails on Node 25)

### Installation

```bash
git clone <repository>
cd Personal-Website
npm install
```

### Configuration

Optional `.env.local` (defaults are baked in for the public project):

```env
NEXT_PUBLIC_SANITY_PROJECT_ID=ubdc9y57
NEXT_PUBLIC_SANITY_DATASET=personal_website
NEXT_PUBLIC_SANITY_API_VERSION=2024-01-01
# Only needed to read drafts / private datasets:
SANITY_READ_TOKEN=...
```

### Development

```bash
npm run dev          # Website + embedded Studio → http://localhost:3000 (/studio)
```

## Scripts

```bash
npm run dev       # Development server
npm run build     # Production build
npm start         # Production server
npm run lint      # ESLint
npm run typegen   # Extract Sanity schema + regenerate sanity/sanity.types.ts
```
